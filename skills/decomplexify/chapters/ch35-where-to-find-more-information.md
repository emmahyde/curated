# Chapter 35: Where to Find More Information

## Core Idea
No single book covers all of software development; McConnell maps the resource landscape by category and provides a tiered reading plan so practitioners can build a complete professional library systematically.

## Key Concepts

- **Construction Resources**: Books focused specifically on the act of writing code — the primary gap McConnell originally wrote Code Complete to fill. Includes books on code construction, refactoring, and coding standards.
- **Topics Beyond Construction**: The surrounding knowledge areas every professional needs: requirements, architecture/design, project management, testing, and process. McConnell recommends specific books in each area (Pressman's *Software Engineering*, Sommerville's *Software Engineering*, Boehm's work on estimation).
- **Periodicals**: Ongoing sources for keeping current. Key publications: *IEEE Software* (practitioner-focused), *IEEE Computer* (broader), *Communications of the ACM* (more academic), *Software Development* magazine (practitioner). McConnell notes that most developers do not read any periodicals — a significant missed opportunity.
- **A Software Developer's Reading Plan**: A structured three-tier plan — introductory, journeyman, and professional — so developers at any level know what to read next rather than facing an undifferentiated heap of titles.
- **Professional Organizations**: IEEE Computer Society (`www.computer.org`) as the most practitioner-oriented; ACM (`www.acm.org`) as the original professional organization, somewhat more academic. Local user groups for language/platform communities.

## Resource Categories McConnell Maps

- **Software construction** (the book's own domain): coding, refactoring, coding standards, code review.
- **Software design and architecture**: patterns (GoF), object-oriented design, software architecture.
- **Software project management**: estimation, scheduling, team dynamics, People-ware topics.
- **Software quality and testing**: testing techniques, inspection, quality assurance.
- **The human side of software**: psychology of programming (Weinberg), professional development, team dynamics.
- **Annotated bibliographies**: *ACM Computing Reviews*; Construx Software's professional development resources at `www.construx.com`.
- **Web resources**: `www.cc2e.com` — McConnell's companion site with updated links organized by chapter.

## Key Takeaways

1. Building a professional library across all knowledge areas (construction, design, management, testing, process) is more valuable than deep expertise in only one area.
2. The three-tier reading plan (introductory → journeyman → professional) gives practitioners a clear progression rather than an overwhelming reading list.
3. Most developers do not read periodicals; those who do have a significant ongoing advantage in staying current with the field.
4. Joining a professional organization (IEEE Computer Society, ACM) connects you to a community of practitioners as dedicated to the profession as you are.
5. The companion site `www.cc2e.com` provides updated, chapter-organized links that extend and refresh the book's references.
