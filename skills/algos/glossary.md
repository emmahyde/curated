# Algorithms Glossary

**Amortized analysis** — average cost per op over a sequence; e.g. resizing-array append is O(1) amortized. (ch01, ch02)
**Approximation algorithm** — runs in poly time with a proven ratio to OPT (e.g. vertex-cover 2-approx). (ch11)
**Augmenting path** — source→sink path in the residual graph with spare capacity; each one raises max-flow. (ch07)
**Backtracking** — DFS over the search space with pruning of dead partial solutions. (ch10)
**Bellman-Ford** — single-source SP tolerating negative edges; O(VE), detects negative cycles. (ch07)
**Big-Oh (O)** — c·g(n) upper-bounds f(n) for n>n₀; suppresses constants. (ch01)
**Big-Omega (Ω)** — c·g(n) lower-bounds f(n); for lower-bound proofs. (ch01)
**Big-Theta (Θ)** — tight bound: both O(g) and Ω(g). (ch01)
**Binary heap** — complete tree in an array; swim/sink; insert & del-max O(lg N), max O(1). (ch05)
**Binary search** — halve a sorted range each step; O(lg N) search, O(N) insert. (ch04)
**Bottom-up DP** — fill the table in dependency order; no call stack, enables rolling-array space. (ch09)
**Boyer-Moore** — substring search skipping via bad-character heuristic; sublinear typical, MN worst. (ch08)
**BST** — O(h) search/insert/delete; h is O(n) unbalanced, O(lg n) balanced. (ch04)
**Bucketsort / distribution sort** — partition by key prefix; O(N) uniform, degrades on skew. (ch03)
**Chaining (separate)** — collisions held in per-slot linked lists; O(1) expected at load α≈1. (ch04)
**Clique** — NP-complete; mutually adjacent vertex set; independent set in the complement. (ch11)
**Cook-Levin theorem** — SAT is NP-complete; every NP problem reduces to SAT in poly time. (ch11)
**DAG shortest path** — relax in topological order; O(E+V); handles negatives; acyclic only. (ch07)
**Dijkstra's algorithm** — greedy SP on non-negative weights; O(E lg V) heap, O(V²) array. (ch07)
**Divide and conquer** — split, recurse, combine; T(n)=2T(n/2)+O(n) → O(n lg n). (ch09)
**Edit distance** — min insert/delete/substitute to transform strings; O(|P|·|T|) DP. (ch09)
**Exchange argument** — greedy-correctness proof: swapping a greedy choice can't improve the solution. (ch10)
**Floyd-Warshall** — all-pairs SP; O(V³)/O(V²); DP over intermediate vertices; no negative cycles. (ch07)
**Hamiltonian cycle** — NP-complete; cycle hitting every vertex once; reduces to TSP. (ch11)
**Hash table** — array indexed by hash(key); O(1) expected ops; no ordering. (ch02, ch04)
**Heapsort** — in-place heap sort; O(N lg N) worst, not stable; ≤2N lg N compares. (ch03, ch05)
**Independent set** — NP-complete; pairwise-non-adjacent vertices; complement of vertex cover. (ch11)
**Indexed priority queue** — PQ with decrease-key in O(lg V); needed by eager Prim and Dijkstra. (ch05, ch07)
**Insertion sort** — shift left into place; ~N²/4 avg, O(N) on sorted input; stable, in-place. (ch03)
**Key-indexed counting** — counting sort using keys as indices; O(N+R); basis of LSD/MSD. (ch08)
**Knuth-Morris-Pratt (KMP)** — DFA substring search; O(M+N), never re-reads text; stream-friendly. (ch08)
**Kruskal's algorithm** — MST via edge-sort + union-find cycle check; ~E lg E. (ch07)
**LCS** — longest common subsequence; edit-distance variant; O(mn) DP. (ch09)
**LIS** — longest increasing subsequence; O(n²) DP, or via LCS of a sorted copy. (ch09)
**Linear partition** — split n ordered items into k groups minimizing max sum; O(kn²) DP. (ch09)
**Linear probing** — open-addressing collisions; probe next slot; O(1) amortized at α≈½. (ch04)
**LSD radix sort** — least-significant-digit first; W counting passes; O(WN), stable. (ch08)
**Max-flow / min-cut** — Edmonds-Karp (BFS augmenting paths) needs O(n³) augmentations; max-flow = min-cut. (ch07)
**Memoization (top-down DP)** — cache recursive results; each subproblem once; call-stack overhead. (ch09)
**Mergesort** — D&C; Θ(N lg N) worst, stable, O(N) aux; optimal compare count. (ch03, ch09)
**MSD radix sort** — most-significant-digit first; recursive; variable-length keys. (ch08)
**NP** — search problems verifiable in poly time (not "non-polynomial"). (ch11)
**NP-complete** — in NP and every NP problem reduces to it; one poly solution collapses P vs NP. (ch11)
**NP-hard** — at least as hard as NP-complete; need not be in NP. (ch11)
**Optimal substructure** — optimal solution contains optimal subproblem solutions; DP precondition. (ch09)
**P** — decision problems solvable in worst-case poly time. (ch11)
**Path compression** — union-find: flatten the find() path to the root; near-optimal with union-by-rank. (ch02, ch07)
**Polynomial-time reduction (A ≤ₚ B)** — poly-time mapping preserving answers; A NPC + A≤ₚB → B NP-hard. (ch11)
**Prim's algorithm** — MST by adding the min crossing edge; lazy O(E lg E), eager O(E lg V). (ch07)
**Quicksort** — partition around a pivot, recurse; ~1.39 N lg N avg, O(N²) worst; in-place, unstable. (ch03)
**3-way quicksort** — lt/eq/gt partition; entropy-optimal; O(N) when keys all equal. (ch03)
**3-way string quicksort** — string D&C; O(N lg N) on distinct keys; avoids MSD's space. (ch08)
**Rabin-Karp** — rolling-hash substring search; Las Vegas or Monte Carlo; O(M+N) typical. (ch08)
**Red-black BST** — balanced BST encoding a 2-3 tree; height ≤2 lg N; all ops O(lg N). (ch04)
**SAT / 3-SAT** — Boolean satisfiability; NPC root (Cook-Levin); 3-SAT caps clauses at 3 literals. (ch11)
**Selection sort** — repeatedly select the min; ~N²/2 compares, exactly N exchanges; unstable. (ch03)
**Shellsort** — insertion sort on h-spaced subarrays; sub-quadratic in practice; avg analysis open. (ch03)
**Simulated annealing** — heuristic accepting worse moves by a cooling schedule; escapes local optima, no guarantee. (ch10)
**Suffix array** — sorted array of all suffixes; O(n) build, O(m) query; compact vs suffix tree. (ch08, ch12)
**Suffix tree** — compressed trie of all suffixes; O(n) build, O(m) query; LCS & matching. (ch08, ch12)
**Topological sort** — linear DAG order with all edges forward; enables O(E+V) DP on DAGs. (ch06)
**Trie (R-way)** — R children per node; O(key length) ops; O(NR) space. (ch08)
**TSP** — NP-complete (decision form); min-cost tour of all vertices; reduces from Hamiltonian cycle. (ch11)
**TST (ternary search trie)** — 3-way branching trie; trie-like speed, far less space; ordered ops. (ch08)
**Union-Find (disjoint sets)** — weighted quick-union O(lg n); with path compression ~O(α(n)) amortized. (ch02, ch07)
**Vertex cover** — NP-complete; vertex set touching every edge; complement is an independent set. (ch11)
**Worst-case complexity** — max steps over inputs of size n; the primary design guarantee. (ch01)
